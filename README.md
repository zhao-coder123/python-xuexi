# python-xuexi

一个面向前端开发者转向 `Python 服务端 + AI 集成` 的学习型仓库。

这个仓库的定位不是收集零散笔记，而是提供一套可执行的路线：

- 先把 `Python 后端 API 基础能力` 打稳
- 再补 `管理后台常见能力`
- 最后把 `Dify / Agent / 模型平台` 接入到真实业务模块里

当前仓库已经包含：

- 一份 `30 天执行清单版` 学习路线文档
- 一份更适合浏览和展示的 `HTML 页面`
- 一套适合当前项目的 `AI 工具选型建议`
- 一份可直接落地的 `最小后台页面蓝图`

## 仓库目标

这份仓库主要解决下面这类学习目标：

1. 已经会 `Vue / React`，想补 Python 服务端
2. 想用 Python 做 `后台 API`
3. 想学习 `MySQL`、鉴权、权限、上传、日志这些后台基础能力
4. 想把 AI 能力接进业务系统，而不是只停留在调用模型 demo
5. 想通过一个完整项目把技术串起来

一句话总结：

> 用一个“AI 内容管理后台”项目，把 Python 后端能力和 AI 集成能力一起补起来。

## 学习主线

这份仓库推荐的技术主线是：

`FastAPI + MySQL + Redis + 管理后台能力 + Dify / Agent / 模型 API`

### 为什么这么选

因为这个路线更适合下面这种背景：

- 已经会前端
- 不想走纯传统 Python Web 学法
- 也不想一上来就扎进复杂 AI 框架
- 希望先把服务端能力做稳，再逐步贴近 AI 应用开发

### 分阶段重点

#### 第 1 阶段：基础 API 能力

- Python 基础够用
- FastAPI 路由与参数
- Pydantic 校验
- 项目分层

#### 第 2 阶段：后台常见能力

- MySQL
- SQLAlchemy
- Alembic
- 登录 / JWT
- 权限
- 分页 / 过滤 / 排序

#### 第 3 阶段：工程化能力

- 上传
- Redis
- 操作日志
- 错误处理
- 测试
- 代码规范

#### 第 4 阶段：AI 集成能力

- 模型 API 接入
- 流式输出
- Dify 接入
- Agent / Workflow 基础理解
- AI 任务记录

### 工具选型建议

针对这个仓库的目标，建议这样使用：

- `Dify` 作为主线，先把 AI 能力快速接进业务模块
- `n8n` 作为辅助，用来做通知、定时任务、外部系统串联
- `LangChain / LangGraph` 作为进阶内容，留给需要深度定制时再上
