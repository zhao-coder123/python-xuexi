# Day 2: Python 后端常见写法

Day 1 你学的是“Python 最基础能跑起来”。

Day 2 要开始进入“后端代码长什么样”。

今天最应该学的，不是更难的语法，而是下面这些后端里几乎天天会见到的内容。

## 1. 类与对象

为什么 Day 2 要学这个：

- 后端经常要表示一个用户、文章、订单、任务
- 这些数据如果只靠零散变量管理，会很乱
- 用类可以把“数据 + 行为”放到一起

你今天至少要知道：

- 什么是类 `class`
- 什么是对象
- 什么是属性
- 什么是方法
- `self` 表示当前对象本身

在 `day2.py` 里你会看到：

- `User` 类
- `LearningTask` 数据类
- `add_task()`、`finish_task()` 这类实例方法

## 2. 类型注解

为什么要学：

- Python 虽然是动态类型语言，但后端项目里很常写类型注解
- 类型注解可以让代码更清晰，也方便编辑器提示
- FastAPI、Pydantic 都会大量使用类型信息

你今天至少要看懂：

- `name: str`
- `minutes: int`
- `def parse_minutes(raw_value: str) -> int:`
- `list[LearningTask]`
- `dict[str, object]`

注意：

- 类型注解本身不是“强制校验器”
- 它主要是帮助你写出更清晰的代码

## 3. 异常处理

为什么要学：

- 后端不可能假设所有输入都合法
- 用户参数错了、文件不存在、JSON 格式错了都很常见
- 所以你必须学会 `try / except`

你今天至少要知道：

- `try`
- `except`
- `else`
- `finally`
- 自定义异常类

在 `day2.py` 和 `config_tools.py` 里你会看到：

- `StudyDataError`
- `ConfigError`

这两个例子很重要，因为后面写 API 时，你会经常把底层错误转换成更清晰的业务错误。

## 4. datetime

为什么要学：

- 后端几乎离不开时间
- 创建时间、更新时间、过期时间、任务执行时间都要处理

你今天至少要会：

- `datetime.now()`
- `timedelta(days=1)`
- `strftime()` 格式化输出
- `isoformat()` 转字符串

## 5. 标准库常用模块

Python 很强的一点是标准库很实用。

Day 2 建议优先认识这些：

- `json`：处理 JSON 数据
- `pathlib`：处理文件路径
- `collections`：例如 `Counter`
- `statistics`：例如 `mean`
- `time`：例如计时
- `datetime`：时间处理
- `asyncio`：异步编程

不用一天全背下来，但至少要先“见过并用过”。

## 6. async / await 和 asyncio

为什么要学：

- FastAPI 是异步友好的框架
- 后面你会写 `async def`
- 你需要先理解异步函数和同步函数的区别

你今天至少要知道：

- `async def` 是定义异步函数
- `await` 是等待异步结果
- `asyncio.run()` 可以运行异步入口
- `asyncio.gather()` 可以同时等待多个异步任务

你不用今天就把异步吃透。

Day 2 的目标只是：

- 能看懂异步示例
- 知道为什么并发等待通常比顺序等待更快

## 你现在还需要额外补充的知识

结合你现在的 Day 1 和 Day 2 内容，建议你额外补下面这些基础点。

### 1. 字符串常见操作

你目前已经见过字符串，但还应该补：

- `.strip()`
- `.split()`
- `.replace()`
- f-string

这是后面处理接口参数时很常用的基础。

### 2. 函数参数的更多写法

你现在会基本函数了，但还应该继续补：

- 默认参数
- 关键字参数
- 位置参数
- 可选参数

因为 FastAPI 参数设计和普通函数参数理解是连着的。

这次我已经在 `day2.py` 里补了一个简单示例，
让你能同时看到：

- 默认参数
- 关键字参数
- 位置参数

### 3. 返回值统一意识

你现在要开始建立一个习惯：

- 一个函数最好有明确输入
- 有明确输出
- 不要到处直接 `print()` 代替返回值

学习阶段可以打印，但后端项目里很多函数最终是给别人调用的，不只是给你自己看。

### 4. 可变对象和不可变对象

这个点你 Day 1 提过，但还需要更明确：

- `list`、`dict`、`set` 是可变的
- `str`、`int`、`tuple` 是不可变的

这个知识后面会直接影响：

- 函数传参
- 默认参数陷阱
- 数据修改是否会影响原对象

### 5. import 路径和模块执行方式

你已经会简单导入了，但最好再补一个认识：

- 直接运行文件
- 作为模块导入
- 相对导入和绝对导入

这个在项目一拆目录时就会遇到。

### 6. JSON 是后端核心基础

从 Day 2 开始，你要逐步建立这个意识：

- 后端接口最常返回 JSON
- 配置文件也常常是 JSON
- 前后端通信会频繁处理 JSON

所以 `json.loads()`、`json.dumps()` 后面一定要继续熟悉。

这次我也已经把最小 JSON 转换示例补进 `day2.py` 了。
