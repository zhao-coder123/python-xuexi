# Day 26: Agent / 工作流视角理解

今天重点：

- 什么是 Agent
- 什么是工具调用
- 什么是工作流
- 适合业务系统的 Agent 场景

## 这次给你的最小示例

`workflow_demo.py` 把一个文章处理流程拆成了：

1. 生成摘要
2. 提取关键词
3. 生成标题

这还不是复杂 Agent，但已经是很好的工作流入门。

## 运行方式

```bash
cd 2026-05-11-day26
python3 workflow_demo.py
```

## 查漏补缺

1. Agent 不等于“会聊天”
2. 很多业务真正需要的是稳定工作流，而不是复杂 Agent
3. 先把多步骤任务拆清楚，再谈工具编排
