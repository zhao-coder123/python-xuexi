# Day 21: Week 3 收口

Day 21 的任务不是再堆新语法，而是把前面的能力收口成一个更像样的后台原型。

## 本日交付目标

- 用户登录
- 权限控制
- 用户管理
- 内容管理
- 上传接口
- 操作日志
- README 基础版
- Swagger 文档可演示

## 建议你今天做的事情

1. 把 Day 12 到 Day 20 的示例重新串起来
2. 确定哪些能力已经有可运行最小版
3. 统一目录命名和模块命名
4. 整理一版对外可展示的 README

## 查漏补缺

Week 3 结束时最容易漏的不是代码，而是整体串联能力：

1. 你是否真的能说清每个模块负责什么
2. 你是否知道哪些能力还是 demo，哪些已经接近正式项目
3. 你是否已经形成“模块化”视角，而不是只会写零散接口
